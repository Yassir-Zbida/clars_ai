# Clars.ai — Impeccable design context

Design guidance for this repo uses [Impeccable](https://github.com/pbakaus/impeccable) (Apache-2.0) via `.cursor/skills/frontend-design` and related command skills. Refine this file over time; run `/teach-impeccable` in Cursor to revisit and expand.

## Design Context

### Users

Freelancers and solopreneurs managing clients, projects, invoices, and cash flow—often switching contexts quickly between sales, delivery, and admin. Primary regions: MENA and Morocco; interfaces may be used in **English, French, or Arabic** (RTL for `ar`).

### Brand Personality

**Professional, calm, capable** — the product should feel like a serious workspace, not a toy or a generic “AI startup” landing page. Trust and clarity matter more than spectacle.

### Aesthetic Direction

- **Tone**: Refined productivity / editorial clarity — strong hierarchy, intentional whitespace, subtle depth where it helps scanning (tables, kanban, finance).
- **Themes**: Support **light and dark**; tint neutrals toward the brand palette rather than flat gray everywhere.
- **Differentiation**: Memorable through **typography pairing**, **consistent rhythm**, and **locale-aware layout** (RTL, number/currency formatting), not through decorative gradients or stock dashboard hero layouts.
- **Anti-references**: Purple/cyan neon-on-dark clichés, identical “icon + title + blurb” card grids, Inter-only stacks, glassmorphism for its own sake, modals when inline or drawer patterns would work.

### Design Principles

1. **Density with breath** — CRM surfaces need information density; use grouping, alignment, and variable spacing (tight within a task, loose between sections) instead of boxing everything in nested cards.
2. **One clear focal path** — Primary actions and next steps should be obvious; demote secondary actions (ghost buttons, links, menus).
3. **Accessible by default** — Contrast, focus states, reduced motion; no gray body text on saturated backgrounds.
4. **Stack-native quality** — Implement in **Next.js 14, Tailwind, shadcn/ui**; prefer design tokens (`globals.css` / CSS variables), `next/font`, and shared primitives (`SectionCard` / `SectionHeader` from `src/app/dashboard/analytics/_components/analytics-page-shell.tsx` for new dashboard-style sections).
5. **Motion with purpose** — Prefer transform/opacity, ease-out curves; respect `prefers-reduced-motion`; avoid bounce/elastic easing for routine UI.

### Technical constraints

- Next.js App Router, Server Components by default; client only when needed.
- Tailwind + shadcn/ui + Radix; use `cn()` for class composition.
- Charts: Recharts (existing); keep chart styling aligned with the same palette and type scale as the rest of the app.
